# 📌 Day 7 — Mini Project: Todo App (React)

## 📖 Опис

Міні-проєкт для закріплення:

* форм (login/register)
* валідації
* React Hook Form
* routing (React Router)
* URL params
* роботи зі станом

---

## 🚀 Завдання

Реалізувати **Todo App з авторизацією та роутингом**

---

## 🧭 Роутинг

* `/` — список задач (тільки для авторизованого користувача)
* `/login` — сторінка логіну
* `/register` — сторінка реєстрації
* `/todo/:id` — сторінка конкретної задачі

---

## 🔐 Авторизація (без бекенду)

* форма логіну та реєстрації
* після успішного входу:

  * зберегти користувача в `localStorage`
* якщо користувач не авторизований:

  * редірект на `/login`

### ⚠️ Обовʼязково:

Реалізувати **Protected Route**

---

## 📝 Todo функціонал

### Список задач (`/`)

* створення задачі (input + button)
* відображення списку
* кожна задача має:

  * title
  * статус (completed / active)
  * toggle статусу
  * delete
  * перехід на детальну сторінку

---

### Детальна сторінка (`/todo/:id`)

* відображення:

  * title
  * статус
* дії:

  * змінити статус
  * повернутись назад

### Обробка помилок:

* якщо `id` не існує:

  * показати `404` або зробити редірект

---

## 💾 Стан і дані

* використовувати `useState`
* зберігати todos в `localStorage`
* при оновленні:

  * дані мають синхронізуватись між сторінками

---

## ⚙️ Обовʼязкові технічні вимоги

### 1. Protected Route

* обмежити доступ до `/` і `/todo/:id`

---

### 2. URL params

* обробити випадки:

  * `/todo/abc`
  * `/todo/999`

---

### 3. Синхронізація стану

* зміни в todo повинні відображатись:

  * і в списку
  * і в деталях

---

### 4. UX

* loading (можна fake через `setTimeout`)
* повідомлення, якщо список пустий

---

## 🔥 Додатково (опціонально)

* фільтр задач (all / completed / active)
* пошук
* logout

---

## 🎯 Результат

Після виконання ти повинен:

* розуміти як працює routing
* вміти працювати з URL params
* контролювати стан між сторінками
* реалізувати базову авторизацію
